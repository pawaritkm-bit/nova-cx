# NOVA Customer Experience System — โจทย์ต้นฉบับ (Original Brief)

> เอกสารนี้คือโจทย์ต้นฉบับจากผู้ว่าจ้าง ใช้เป็น single source of truth ของทีม
> ห้ามแก้เนื้อหาส่วนนี้ — บันทึกไว้เพื่ออ้างอิง

## บทบาทที่ต้องสวม
CTO, Product Manager, UX/UI Designer, LINE OA Developer, LIFF Developer, Database Architect, AI Engineer, Customer Experience Specialist และผู้เชี่ยวชาญระบบสำนักงานบัญชีในประเทศไทย

## PROJECT NAME
**NOVA Customer Experience System** — ระบบประเมินความพึงพอใจและติดตามคุณภาพบริการของ Finovas Accounting ผ่าน LINE OA

## เป้าหมายของระบบ
LINE OA ที่ใช้งานสะดวก เป็นมิตร แบบผู้ช่วยในแชท (Branding/UI/ตัวละครออกแบบใหม่ทั้งหมด ห้ามลอกแบรนด์อื่น) ใช้สำหรับ:
1. ประเมินความพึงพอใจต่อสำนักงานบัญชีทุก 3 เดือน
2. ประเมินความพึงพอใจต่อนักบัญชีที่ดูแลลูกค้าทุกเดือน
3. แยกประเมินหัวหน้าทีมบัญชีและลูกทีมบัญชี
4. ประเมินความพึงพอใจต่อเซลล์ ทั้งขายได้และขายไม่ได้
5. ตรวจจับปัญหา/ความเสี่ยงก่อนลูกค้ายกเลิกบริการ
6. แจ้งเตือนหัวหน้า/ผู้บริหารให้แก้ไขปัญหา
7. ใช้ AI สรุปความคิดเห็นและแนะนำแนวทางแก้ไข
8. Dashboard วัดคุณภาพสำนักงาน ทีมบัญชี และทีมขาย
9. **ห้าม AI รับปากเรื่องค่าชดเชย/คืนเงิน/ลดราคา/ผลลัพธ์แทนบริษัทโดยอัตโนมัติ**

- ต้องแยกข้อความทั่วไปออกจากเรื่องเร่งด่วน

## MASCOT + LOADING ANIMATION
Mascot: การ์ตูน "นกอินทรี" ใส่แว่น ใส่สูทน้ำเงินเข้ม/สีแบรนด์ ถือเครื่องคิดเลข ฉลาด คล่องแคล่ว เป็นมิตร น่าเชื่อถือ ไม่ดุ/น่ากลัว ไม่เหมือนแบรนด์อื่น — ต้องมี Full Character, Icon, Profile, Loading Animation
Loading (ระหว่างโหลด LIFF Form): นกอินทรีวิ่ง เครื่องคิดเลขขยับ เอกสาร/ใบเสร็จปลิว วนซ้ำ 1–2 วิ โหลดเร็วบนมือถือ รองรับ Lottie/SVG/WebP มีข้อความสลับ ถ้าโหลด >5 วิ แสดงปุ่ม "ลองใหม่" + "แจ้งปัญหา"

## ช่องทางเข้าแบบประเมิน
Rich Menu, Flex Message, ลิงก์ LIFF, QR Code, แจ้งเตือนอัตโนมัติ, พนักงานส่งให้, ส่งหลังเปลี่ยนสถานะขาย, ส่งตามรอบบริการ
Rich Menu: ประเมินการบริการ / แจ้งปัญหาด่วน / ติดต่อทีมดูแล / ตรวจสอบสถานะเรื่องร้องเรียน / คำถามที่พบบ่อย / นโยบายข้อมูลส่วนบุคคล

## แบบประเมิน 4 ประเภท (ส่งถูกประเภท ไม่ส่งซ้ำในรอบเดียวกัน)
- **A** สำนักงานบัญชี ทุก 3 เดือน
- **B** ทีมบัญชี ทุกเดือน (แยกหัวหน้าทีม/ลูกทีม)
- **C** เซลล์ กรณีขายได้ (Won)
- **D** เซลล์ กรณีขายไม่ได้ (Lost) — สั้นกว่าแบบอื่น

### FORM A (สำนักงาน/3 เดือน)
ส่วน1 ข้อมูลอ้างอิง (auto-fill ถ้ามี): รหัสลูกค้า, ชื่อลูกค้า, ชื่อกิจการ, ประเภทบริการ, วันเริ่มบริการ, ทีมบัญชีที่รับผิดชอบ, รอบประเมิน, ช่องทางที่ได้รับ — ห้ามให้กรอกซ้ำโดยไม่จำเป็น
ส่วน2 ความพึงพอใจภาพรวม คะแนน 1–5 (1=ไม่พอใจมาก…5=พอใจมาก) 10 ข้อ: ภาพรวม, ความถูกต้องบัญชี/ภาษี, ความรวดเร็ว, ความชัดเจนการอธิบาย, การแจ้งกำหนดส่งเอกสาร/ภาษี/งานสำคัญ, การติดตาม/แจ้งสถานะ, ความสะดวกติดต่อ, ความเหมาะสมค่าบริการ, ความมั่นใจในการดูแล, การรับผิดชอบ/แก้ไขเมื่อมีปัญหา
ส่วน3 ปัญหาที่เคยพบ (เลือกได้หลายข้อ) — มีตัวเลือก "ยังไม่พบปัญหา" ที่ต้องเลือกเดี่ยว (เลือกพร้อมข้ออื่นไม่ได้); ตัวเลือกรวม: ติดต่อยาก, ตอบช้า, ไม่ทราบสถานะ, ส่งข้อมูล/เอกสารซ้ำ, แจ้งเอกสารไม่ครบ/ไม่ชัด, แจ้งงาน/กำหนดช้า, งานล่าช้า, งานผิด/แก้หลายครั้ง, อธิบายไม่เข้าใจ, ข้อมูลไม่ตรงกันจากแต่ละคน, ไม่เข้าใจขอบเขตบริการ, ค่าใช้จ่ายไม่เข้าใจ, ค่าบริการไม่เหมาะสม, ไม่ได้คำแนะนำเชิงรุก, เปลี่ยนผู้ดูแลบ่อย, กังวลความปลอดภัยข้อมูล, กังวลความถูกต้อง/กำหนดภาษี, ยังไม่พบปัญหา, อื่นๆ
ส่วน4 ความต้องการเพิ่มเติม: 4 คำถามปลายเปิด + ตัวเลือกบริการเสริม (วางแผนภาษี, บัญชีภายใน, วางระบบบัญชี, Internal Control, รายงานผู้บริหาร, CFO/วางแผนการเงิน, Audit, Holding, Due Diligence, จดทะเบียน/แก้ไขบริษัท, กฎหมายธุรกิจ, การตลาด, AI/ระบบอัตโนมัติ, ยังไม่ต้องการ, อื่นๆ)
ส่วน5 Loyalty/ความเสี่ยงยกเลิก: NPS 0–10, แนวโน้มใช้ต่อ (ใช้ต่อแน่นอน/มีแนวโน้มใช้ต่อ/ยังไม่แน่ใจ/กำลังเปรียบเทียบ/มีแนวโน้มยกเลิก/ต้องการยกเลิก), เหตุผลที่อาจไม่ใช้ต่อ, สิ่งที่ควรแก้ทันที
ส่วน6 การติดต่อกลับ: (ไม่ต้อง/ทีมดูแล/หัวหน้าทีม/ผู้บริหาร/ด่วน) + ช่องทาง (LINE/โทร/อีเมล) + ช่วงเวลาสะดวก

### FORM B (นักบัญชี/รายเดือน)
บัญชีลูกค้าหนึ่งอาจมีหัวหน้าทีม + ลูกทีม + ผู้ประสานงานหลายคน — ต้องแสดงรูป/ชื่อเล่น/ตำแหน่ง/บทบาท กันประเมินผิดคน
เลือก: ประเมินหัวหน้าทีม / ลูกทีม / ทั้งทีม / จำไม่ได้ว่าใครดูแล
หัวหน้าทีม 7 ข้อ (1–5): คำแนะนำ/ตัดสินใจ, ตรวจงานทีมก่อนส่ง, รับผิดชอบ/ติดตามเมื่อมีปัญหา, อธิบายความเสี่ยงเข้าใจง่าย, คำแนะนำเชิงรุก, คุมคุณภาพ/กำหนดเวลาทีม, ความมั่นใจในหัวหน้าทีม
ลูกทีม 10 ข้อ (1–5): ตอบเร็ว, สุภาพ/มืออาชีพ, แจ้งเอกสารครบ/ชัด, ติดตามเหมาะสมไม่รบกวนเกิน, แจ้งสถานะต่อเนื่อง, ทำงานถูก/ลดขอซ้ำ, อธิบายเข้าใจง่าย, แก้ปัญหาเบื้องต้น, ส่งต่อหัวหน้าเมื่อเกินขอบเขต, พึงพอใจโดยรวม
ปลายเปิด: ทำได้ดี, ต้องปรับปรุง, เรื่องที่ยังไม่ได้แก้, ต้องการเปลี่ยนผู้ดูแลหรือไม่/เพราะอะไร
ปัญหาเฉพาะนักบัญชี: ตอบช้า, ไม่ตอบ, อ่านแล้วไม่ตอบ, ไม่สุภาพ, อธิบายไม่เข้าใจ, แจ้งเอกสารไม่ครบ, ขอเอกสารซ้ำ, ไม่แจ้งกำหนด, ไม่แจ้งสถานะ, งานล่าช้า, งานผิด, ข้อมูลไม่ตรงกัน, แก้ปัญหาไม่ได้, ไม่ส่งต่อหัวหน้า, เปลี่ยนผู้ดูแลบ่อย, ไม่มีปัญหา, อื่นๆ

### FORM C (เซลล์ ขายได้/Won)
ส่งหลังชำระเงิน/ยืนยันซื้อ เว้น 1–3 วัน; เซลล์ห้ามเลือกส่งเฉพาะลูกค้าที่พอใจ
10 ข้อ (1–5): ตอบเร็ว, เข้าใจปัญหา/ความต้องการ, อธิบายบริการ/ขอบเขตชัด, อธิบายราคา/ค่าใช้จ่าย/เงื่อนไขครบ, แนะนำเหมาะสมไม่ขายเกิน, ตอบตรงประเด็น/ถูกต้อง, ติดตามเหมาะสมไม่กดดัน, สิ่งที่แจ้งตรงกับใบเสนอราคา/บริการจริง, ส่งต่อทีมบริการราบรื่น, พึงพอใจเซลล์โดยรวม
เพิ่มเติม (ปลายเปิด): เหตุผลเลือก Finovas, เรื่องที่ยังไม่เข้าใจ/กังวลก่อนซื้อ, เซลล์รับปากเกินใบเสนอราคาหรือไม่, ค่าใช้จ่าย/เงื่อนไขที่ไม่ได้แจ้งล่วงหน้า, ทำได้ดี, ควรปรับปรุง
เหตุผลตัดสินใจซื้อ (choice): น่าเชื่อถือแบรนด์, เซลล์แนะนำดี, ตอบเร็ว, ราคาเหมาะสม, ขอบเขตครบ, เข้าใจธุรกิจ, มีผู้แนะนำ, เห็นจากคอนเทนต์, โปรโมชั่น, ต้องการเริ่มด่วน, อื่นๆ

### FORM D (เซลล์ ขายไม่ได้/Lost) — สั้น
ส่งเมื่อปฏิเสธ/ปิด Lost/ไม่ตอบหลังติดตามครบเกณฑ์
ข้อความเปิด: "แม้ครั้งนี้เราอาจยังไม่ได้ดูแลคุณ น้อง NOVA ขอถามสั้นๆ เพื่อให้ Finovas ปรับปรุงการให้คำแนะนำได้ดีขึ้นนะคะ"
5 คำถามหลัก: พึงพอใจคำแนะนำเซลล์, เซลล์เข้าใจปัญหา/ความต้องการ, อธิบายบริการ/ราคา/ขอบเขตชัด, ติดตามมาก/น้อย/เหมาะสม, เหตุผลหลักที่ยังไม่ตัดสินใจ
เหตุผล (choice): ราคาสูงกว่างบ, เปรียบเทียบเจ้าอื่น, เลือกเจ้าอื่นแล้ว, ยังไม่พร้อมเริ่ม, ยังไม่เห็นจำเป็น, ขอบเขตไม่ตรง, ไม่มั่นใจบริการ, ไม่เข้าใจสิ่งที่ได้รับ, ระยะเวลาไม่ตรง, ติดต่อ/ตอบช้า, เซลล์ติดตามมากเกิน, เซลล์ไม่ติดตามต่อ, ข้อมูลไม่ครบ, ราคาไม่ชัด, มีค่าใช้จ่ายเพิ่ม, ขออนุมัติผู้มีอำนาจไม่ผ่าน, ปัญหาแก้แล้ว, ติดต่อใหม่ภายหลัง, ไม่ต้องการรับติดต่ออีก, อื่นๆ
6. สิ่งที่ Finovas ปรับแล้วจะพิจารณาใหม่, 7. อนุญาตติดต่อในอนาคต (ทันที/ภายใน7วัน/ภายใน30วัน/เมื่อมีข้อเสนอตรงใจ/ไม่ต้องการรับติดต่ออีก)
**หากเลือก "ไม่ต้องการรับติดต่ออีก" → บันทึก Do Not Contact + หยุด Automation การขายทันที**

## Conditional Logic
- คะแนน 4–5: ถามสิ่งที่ทำได้ดี
- คะแนน 3: ถามเรื่องที่อยากให้ปรับปรุง
- คะแนน 1–2: เปิดคำถามหาสาเหตุ + ตัวเลือกให้ติดต่อกลับ
- งานผิด → ถามประเภทงาน/วันที่/รายละเอียด; ตอบช้า → ถามรอนานเท่าใด; ค่าใช้จ่ายไม่ชัด → ถามรายการใด
- ขอเปลี่ยนผู้ดูแล → เข้ากระบวนการตรวจสอบโดยหัวหน้าทีม; ขอยกเลิก → สร้าง Retention Case ทันที
- keyword สำคัญ: "สรรพากร, ค่าปรับ, เสียหาย, ฟ้อง, ข้อมูลรั่ว, เอกสารหาย, ยกเลิก, คืนเงิน" → จัดเป็นเรื่องสำคัญให้มนุษย์ตรวจ **แต่ AI ห้ามตัดสินจาก keyword อย่างเดียว ต้องวิเคราะห์บริบทด้วย**

## ระดับความเร่งด่วน + แจ้งเตือน
- **Critical**: คะแนนต่ำมาก / จะยกเลิก / เสียหายภาษี-การเงิน / ข้อมูลรั่ว-เอกสารหาย / ร้องเรียน-ดำเนินคดี / กล่าวหาทุจริต → แจ้งผู้บริหาร+หัวหน้าฝ่ายทันที
- **High**: คะแนนหัวข้อสำคัญ 1–2 / ขอเปลี่ยนนักบัญชี / งานผิด / ล่าช้ากระทบกำหนด / เซลล์แจ้งไม่ตรงบริการจริง → แจ้งหัวหน้าทีมภายในวันทำการเดียวกัน
- **Medium**: คะแนน 3 / ขอปรับการสื่อสาร / ตอบช้าแต่ไม่กระทบงาน → สร้าง Task ติดตาม
- **Positive**: คะแนน 4–5 → เก็บ Positive Feedback / ขออนุญาตทำ Testimonial ภายหลัง (ห้ามใช้โดยไม่ยินยอม)

ทุก Case ต้องมี: Case ID, เจ้าของเคส, ระดับเร่งด่วน, กำหนดเวลาตอบกลับ, สถานะ, บันทึกการติดต่อ, วิธีแก้ไข, วันปิดเคส, ความพึงพอใจหลังแก้ไข
สถานะ: New, Acknowledged, Investigating, Waiting for Customer, In Progress, Resolved, Closed, Reopened

## แจ้งเตือนตามรอบ
- สำนักงาน: ทุก 3 เดือนจากวันเริ่มบริการ (หรือรอบกลางรวม), เตือนซ้ำ ≤2 ครั้งมีระยะห่าง, หยุดเมื่อประเมินแล้ว/ยกเลิก/ขอไม่รับ
- นักบัญชี: เดือนละครั้ง ไม่ถี่เกิน, ไม่มี interaction เลื่อนได้, เปลี่ยนผู้ดูแลต้องประเมินตรงคนที่ดูแลจริงช่วงนั้น
- เซลล์: Won หลังยืนยันซื้อ, Lost หลังปิดสถานะ, No Response เมื่อครบเกณฑ์, ห้ามส่งซ้ำในดีลเดียว

- Backend: API-first, Node/TS, REST, Background Jobs+Scheduler, LINE Webhook, Queue สำหรับแจ้งเตือน+AI

- Integration: LINE OA/Messaging API/LIFF, CRM/ฐานลูกค้า, Email/LINE Notify ทีมภายใน, AI provider แบบสลับได้, Webhook
- Monitoring: Error Logging, Audit Log, Delivery Status, Failed Job Retry, Rate Limit, Health Check, Analytics Event

## PDPA
แจ้งก่อนเริ่ม: วัตถุประสงค์, ประเภทข้อมูล, ผู้เข้าถึง, ระยะเวลาเก็บ, ช่องทางใช้สิทธิ, การใช้ AI วิเคราะห์, การขออนุญาตติดต่อกลับ
ระบุตัวตน: ส่งแบบระบุตัวตน (ติดตามแก้ไขได้) / ส่งแบบจำกัดการแสดงชื่อแก่ผู้ถูกประเมิน
**หมายเหตุ: Admin ที่ได้รับอนุญาตอาจเข้าถึงตัวตนได้ในกรณีร้องเรียน/ความปลอดภัย ต้องแจ้งลูกค้าโปร่งใส ห้ามอ้าง Anonymous 100% ถ้ายังเชื่อมกลับถึงตัวบุคคลได้**
