ℹ️ ทุกการเปลี่ยนแปลง (เพิ่มนักบัญชี / เปลี่ยนหัวหน้าบัญชี / ย้ายทีม) เป็นแบบ มีผลตามวันที่ (effective-dated) — ระบบจริงจะเก็บประวัติเดิมไว้ (ปิดด้วย valid_to) เพื่อให้ Feedback ย้อนหลังผูกกับผู้ดูแลที่ดูแลจริง ณ ช่วงเวลานั้น
＋ สร้างทีมบัญชีใหม่ mock — ไม่บันทึกจริง

เปิดทีมใหม่พร้อมกำหนดหัวหน้าบัญชี (และนักบัญชีเริ่มต้นถ้ามี) — ทีมใหม่จะมีปุ่มจัดการเหมือนทีมอื่น

mock — ระบบจริงบันทึกลง teams + team_members แบบ effective-dated (เก็บประวัติ)

🕓 ประวัติการเปลี่ยนแปลงโครงสร้างทีม (effective-dated)
ยังไม่มีการเปลี่ยนแปลงในเซสชันนี้ (ลองกดปุ่มในการ์ดทีมด้านบน)
🔗 นี่คือข้อมูลที่แบบประเมิน B (ประเมินนักบัญชี) ดึงไปผูกให้ลูกค้าโดยอัตโนมัติ — ลูกค้าไม่ต้องเลือกเองว่าจะประเมินใคร ระบบจะ snapshot หัวหน้าบัญชี + นักบัญชีที่ผูกอยู่ ณ วันส่งแบบประเมิน แล้วสร้างชุดคำถามต่อคนให้เอง (กันประเมินผิดคน)
📋 บริษัทและผู้ดูแลปัจจุบัน
บริษัททีมหัวหน้าบัญชีนักบัญชีที่ดูแลมีผลตั้งแต่
➕ ผูกผู้ดูแลใหม่ให้บริษัท mock — ไม่บันทึกจริง
เมื่อบันทึก การผูกเดิมของบริษัทจะถูกปิด (valid_to) และเก็บเป็นประวัติ
🕓 ประวัติการผูกผู้ดูแล (valid_to)
ยังไม่มีการเปลี่ยนแปลงในเซสชันนี้
📌 ระบบจะส่งแบบประเมินที่ถูกต้องให้ลูกค้าแต่ละคนอัตโนมัติ ตามรอบ/สถานะ — ลูกค้าไม่ได้เลือกชนิดแบบประเมินเอง และจะเห็นเพียงแบบเดียวของตัวเองผ่านลิงก์ invitation. หน้านี้ไว้ให้ทีมพรีวิว/ตรวจสอบแต่ละชนิดในมุมมองลูกค้า
⚙️ กติกาการส่งอัตโนมัติ
  • ระบบเชื่อม 2 LINE OA: 💙 OA Care (ประเมินสำนักงาน + พนักงานทีมบัญชี) และ 🤝 OA Sale (ประเมินเซล ขายได้/ขายไม่ได้) — ลูกค้าจะได้รับแบบประเมินผ่าน OA ที่ตรงกับเรื่องนั้น
  • หลังลูกค้าแอด LINE OA เป็นเพื่อน → ระบบส่งแบบประเมินเข้าแชทให้อัตโนมัติ (เริ่มที่สำนักงาน) แล้วเข้ารอบส่งต่อไป
  • สำนักงาน = ทุก 3 เดือน · นักบัญชี = ทุกต้นเดือน · เซล = ครั้งเดียวจบต่อ 1 ดีล (ไม่วนซ้ำ)
  • แบบประเมินสำนักงาน = ส่งเข้ากลุ่ม LINE (มีนักบัญชี+ลูกค้า) ทุก 3 เดือน — ประเมินภาพรวมสำนักงาน
  • แบบประเมินนักบัญชี (ประเมินลูกน้อง) = ส่งเข้าแชตส่วนตัวลูกค้า ทุกต้นเดือน — เป็นการประเมินตัวบุคคล จึงส่งส่วนตัวเพื่อให้ลูกค้าตอบตรงไปตรงมา (นักบัญชีไม่เห็น)
  • ระบบส่งเองอัตโนมัติจากรอบ/สถานะจริง — พนักงาน/ลูกค้าไม่ได้เลือกส่งเอง; มีกันส่งซ้ำในรอบ/ดีลเดียว
